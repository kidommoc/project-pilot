# Contract: Workflow Transition Actions

**Opened**: 2026-04-04T23:46 | **Priority**: high

## Goal

为 project-pilot 的完整 workflow 添加 Phase 转换动作（Transition Actions），明确每个 phase 切换时必须执行的操作。

## Problem Statement

当前 workflow 只定义了 Exit Criteria（流转条件），缺少 Transition Actions（流转动作）。这导致：
- Phase 切换时不清楚需要执行哪些具体操作
- README 更新时机不明确
- Git 操作的时机和责任边界模糊

## Proposal

为 P1→P2、P2→P3、P3→P4、P4→Done 四个转换点补充明确的 Transition Actions，包括：
- 文件状态变更（archive、README 更新）
- Commit 时机和内容
- Human confirmation 的触发点
- Git 打包执行的流程（P4）

## Acceptance Criteria

- [ ] workflow-phase1.md 更新：添加 P1→P2 Transition Actions
- [ ] workflow-phase2.md 更新：添加 P2→P3 Transition Actions
- [ ] workflow-phase3.md 更新：添加 P3→P4 Transition Actions
- [ ] workflow-phase4.md 更新：添加 P4→Done Transition Actions（含 git 打包执行流程）

## Implementation Plan

1. 更新 `references/workflow-phase1.md`
   - 在 Exit Criteria 前添加 "Transition Actions (P1→P2)" 章节
   - 包含：archive 000、update README Phase→2、commit `iteration: v{x.x.x} ready`

2. 更新 `references/workflow-phase2.md`
   - 在 Exit Criteria 前添加 "Transition Actions (P2→P3)" 章节
   - 包含：确认 contracts archived、update README Phase→3、commit `docs: phase 2 → 3`

3. 更新 `references/workflow-phase3.md`
   - 在 Exit Criteria 前添加 "Transition Actions (P3→P4)" 章节
   - 包含：generate audit report、human confirms、update README Phase→4、commit `docs: phase 3 → 4`

4. 更新 `references/workflow-phase4.md`
   - 在 Exit Criteria 前添加 "Transition Actions (P4→Done)" 章节
   - 包含：prepare README cleanup、plan release actions、human confirms、claw executes all

## Boundary

- **Touches**: `references/workflow-phase{1,2,3,4}.md`
- **Does NOT touch**: `contracts/lightweight-workflow.md`（轻量工作流保持不变）
