# Contract: force-workflow-read
**Opened**: 2026-04-06 | **Priority**: high

## Goal
Fix SKILL.md Session Startup to mandate reading workflow files before execution.

## Problem Statement
Current SKILL.md Session Startup only reads README.md to determine Phase, then proceeds based on Phase. Critical execution mechanisms in workflow-phase*.md (e.g. Phase 2's A/B/C Gates, squash commits verification) are never explicitly read, causing Claw to skip Phase ordering and forget squash commits during contract closure.

## Proposal
Modify SKILL.md Session Startup:

1. **Add explicit workflow read step**:
   ```
   2. Read `references/workflow-phase{N}.md` where N = current Phase
   ```

2. **Add ⛔ mandatory marker**:
   ```
   ⛔ Rule: Do not proceed without reading the workflow file
   ```
