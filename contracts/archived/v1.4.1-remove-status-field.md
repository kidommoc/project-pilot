# Contract: 移除 Contract 模板中的 Status 字段

**Opened**: 2026-03-29T10:10  
**Priority**: low  
**Type**: Meta (project-pilot 自身改进)

---

## Goal

移除 Contract 模板文件头中的 Status 字段，因为合同所属目录已表明其状态，避免冗余和一致性风险。

---

## Problem Statement

**现状**：
- Contract 模板文件头包含 `**Status**: draft | in_progress | completed` 或 `**Current**: ...` 字段
- 合同实际状态由所在目录决定（`draft/`, `open/`, `in_progress/`, `archived/`）
- 文件内状态字段与目录状态可能不一致，造成混淆

**后果**：
- 维护负担：移动合同时需要同时更新文件内状态
- 一致性风险：文件内状态可能过时，与目录状态冲突
- 冗余信息：目录已经是真相来源

---

## Proposal

移除所有模板和文档中关于 Contract 文件头 Status 字段的描述和示例。

**修改范围**：
1. `references/templates/mini-contract.md` - 移除 `**Status**: ...` 行
2. `references/templates/contract.md` - 移除 `**Status**: ...` 行
3. `SKILL.md` - 移除 Contract Structure 示例中的 Status section，更新 Session Recovery 算法描述
4. `contracts/lightweight-workflow.md` - 移除 Contract Template 中的 `**Current**: ...` 字段
5. `references/workflow.md` - 移除 "Mark Status as completed" 描述，更新目录引用
6. `contracts/archived/*.md` - 移除历史合同中的 Status 字段

**保留内容**：
- `**Opened**: YYYY-MM-DDTHH:MM` - 保留，用于追踪合同创建时间
- `## Close Contract` 章节 - 保留，用于记录关闭时的验证清单
